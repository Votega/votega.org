# Congressional Legislation & Voting — Feature Design

Adds a curated bill section to each Congressional member page showing sponsored/cosponsored
legislation, the member's vote on key bills, party vote tallies, and links to full bill text.

---

## Goals

- Show each member's vote on a maintainer-curated list of key bills
- Display party tally (D vs R) for each bill so users can see partisan dynamics
- Link to full bill text and official Congress.gov page
- Daily freshness — data pre-fetched at build time, no live API calls from the browser
- No new API keys — everything uses the existing Congress.gov API key

---

## Data Flow

```
curated-bills.json  (maintainer-controlled list of bill IDs)
        ↓
GitHub Actions workflow (update-curated-bills.yml)
        ↓  fetches per bill:
        │  - Bill details, summary, text link   → Congress.gov API
        │  - Roll call number                   → Congress.gov /bill/.../actions
        │  - House party tally + member votes   → Congress.gov /house-vote/.../members
        │  - Senate party tally + member votes  → senate.gov XML (URL from actions)
        ↓
assets/data/curated-bill-votes.json  (generated, never edited manually)
        ↓
member.html  (loads once, filters client-side by bioguideId)
```

## Maintainer Files

### `assets/data/curated-bills.json`

The list you control. Add or remove bills here; the workflow fetches everything else.

```json
{
  "_note": "Curated list of key bills. The workflow fetches vote data for each. Use rollCallOverride to manually specify a roll call number if auto-detection picks the wrong vote.",
  "federal": [
    {
      "congress": 119,
      "type": "s",
      "number": 331,
      "summaryOverride": "Optional plain-English description if the CRS summary is absent or too dense.",
      "rollCallOverride": {
        "house": { "session": 1, "rollNumber": 166 },
        "senate": { "session": 1, "rollNumber": 127 }
      }
    },
    {
      "congress": 118,
      "type": "hr",
      "number": 1319
    }
  ]
}
```

Field notes:
- `summaryOverride` — optional; used when the API returns no summary or the CRS text is too dense for a general audience
- `rollCallOverride` — optional; used when auto-detection picks the wrong vote (e.g. procedural instead of final passage). Omit if auto-detection is reliable.

## API Endpoints Used

| Purpose | Endpoint | Format |
|---|---|---|
| Bill details (title, status, date) | `GET /v3/bill/{congress}/{type}/{number}` | JSON |
| Bill summary | `GET /v3/bill/{congress}/{type}/{number}/summaries` | JSON — take last item |
| Full text link | `GET /v3/bill/{congress}/{type}/{number}/text` | JSON — take last item, `Formatted Text` URL |
| Roll call number | `GET /v3/bill/{congress}/{type}/{number}/actions` | JSON — find `recordedVotes` in actions |
| House party tally + vote meta | `GET /v3/house-vote/{congress}/{session}/{rollNumber}` | JSON |
| House member votes | `GET /v3/house-vote/{congress}/{session}/{rollNumber}/members` | JSON |
| Senate member votes + tally | URL from `recordedVotes[].url` in actions | XML — parse with `xml.etree.ElementTree` |

All JSON endpoints use the existing `CONGRESS_API_KEY` secret. Senate XML is public, no key required.

## Roll Call Auto-Detection Logic

The workflow finds the final passage roll call by scanning bill actions for specific action codes:

| Chamber | Action Code | Meaning |
|---|---|---|
| House | `H37100` | On Passage (House floor record) |
| House | `8000` | Passed/agreed to in House (Library of Congress record) |
| Senate | `17000` | Passed/agreed to in Senate (Library of Congress record) |

If `rollCallOverride` is set in `curated-bills.json`, it skips auto-detection entirely.

---

## Senate Member ID Resolution

Senate XML uses `lis_member_id` (e.g. `S341`), not `bioguideId`. The workflow resolves
members using `last_name` + `state` — unambiguous for senators since no two sitting senators
share a last name from the same state.

The member page uses the same `last_name:state` key to look up a senator's vote at runtime.

When Congress.gov adds Senate vote endpoints to their API (in progress as of 2026), this
can be replaced with a direct `bioguideId` lookup and the stored key format updated.

## Workflow: `update-curated-bills.yml`

New workflow, runs daily alongside the existing member data workflows.

```
Trigger: daily schedule + workflow_dispatch

Steps:
1. Read curated-bills.json
2. For each bill:
   a. Fetch bill details → title, status, statusDate, congressUrl
   b. Fetch summaries   → take last item text (strip HTML); use summaryOverride if set
   c. Fetch text        → take last item Formatted Text URL
   d. Fetch actions     → find recordedVotes by action code (or rollCallOverride)
   e. For each chamber with a recorded vote:
      - House: fetch /house-vote/.../  + /house-vote/.../members → party tally + memberVotes
      - Senate: fetch XML from recordedVotes[].url → parse, compute party tally, build memberVotes
3. Write curated-bill-votes.json
4. Commit and push
```

## Member Page Display

New section on `member.html` below contact info.

**Bill card shows:**
- Title (linked to `congressUrl`)
- Summary (truncated with expand option if long)
- Status + date
- Member's vote — `Aye` / `No` / `Not Voting` / `Present` / `No vote recorded`
- Party tally bar — visual D vs R breakdown (yea votes)
- "Read full text" link

**Matching logic (client-side):**
```javascript
// House member
const myVote = bill.votes.house?.memberVotes[member.bioguideId];

// Senate member
const key = `${member.lastName}:${member.state}`;
const myVote = bill.votes.senate?.memberVotes[key];
```

## Vote Terminology

Congressional votes use specific terminology — display it accurately:

| Raw value | Display |
|---|---|
| `Aye` / `Yea` | Yes |
| `No` / `Nay` | No |
| `Present` | Present (show-but-no-vote) |
| `Not Voting` / `Absent` | Not Voting |
| *(missing from memberVotes)* | No vote recorded |

## Risks and Mitigations

| Risk | Mitigation |
|---|---|
| Auto-detection picks wrong roll call | `rollCallOverride` in curated-bills.json |
| Bill has no CRS summary | `summaryOverride` in curated-bills.json |
| Senate XML structure changes | URL comes from bill actions — if format changes, workflow fails loudly rather than silently |
| Member not found in vote record (absent, different term) | Show "No vote recorded" gracefully |
| Congress.gov API rate limits | Workflow runs once daily; number of curated bills kept reasonable (target: 10–20) |
| Senate LIS→bioguide resolution fails (name collision) | No two sitting senators share last name + state; log warning if match fails |
