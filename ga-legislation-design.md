# Georgia Legislation & Voting — Feature Design

Adds a curated bill section to each Georgia General Assembly member page showing the
member's vote on key state bills, party vote tallies, and links to full bill text.

---

## Goals

- Show each member's vote on a maintainer-curated list of key GA bills
- Display party tally (D vs R) for each bill so users can see partisan dynamics
- Link to full bill text (PDF) and Open States bill page
- Daily freshness — data pre-fetched at build time, no live API calls from the browser
- No new API keys — everything uses the existing `OPENSTATES_API_KEY`

---

## Data Flow

```
curated-ga-bills.json  (maintainer-controlled list of bill identifiers)
        ↓
GitHub Actions workflow (update-curated-ga-bills.yml)
        ↓  fetches per bill:
        │  - Bill record + votes + abstracts + versions  → Open States /bills search
        │  - Select final passage vote per chamber       → last ['passage'] vote event
        │  - Derive party tally                          → join voter.id → ga-members.json
        ↓
assets/data/curated-ga-bill-votes.json  (generated, never edited manually)
        ↓
ga-member.html  (loads once, filters client-side by OCD person ID)
```

## Maintainer Files

### `assets/data/curated-ga-bills.json`

The list you control. Add or remove bills here; the workflow fetches everything else.

```json
{
  "_note": "Curated list of key GA bills. The workflow fetches vote data for each. Use voteOverride to manually specify a roll call number if auto-detection picks the wrong vote.",
  "ga": [
    {
      "session": "2025-2026",
      "identifier": "HB 90",
      "summaryOverride": "Optional plain-English description if the abstract is absent or too dense."
    },
    {
      "session": "2025-2026",
      "identifier": "SB 34",
      "voteOverride": {
        "house": 56,
        "senate": 239
      }
    }
  ]
}
```

Field notes:
- `session` — the legislative session string as it appears in Open States (e.g. `"2025-2026"`). Check the Open States bill page for the correct value.
- `identifier` — the bill number exactly as it appears in the GA legislature (e.g. `"HB 90"`, `"SB 34"`, `"HR 12"`).
- `summaryOverride` — optional; used when the abstract is absent or too dense for a general audience.
- `voteOverride` — optional; used when auto-detection picks the wrong vote (e.g. a substitute vote instead of final passage). The number is the roll call number from the motion text (`"House Vote #56"`). Omit if auto-detection is reliable.

**How to find a bill's session string:** Search for the bill on [openstates.org](https://openstates.org/ga/bills/) and look at the session listed on the bill page.

Notes on the shape:
- `votes.house.memberVotes` — keyed by OCD person ID (e.g. `"ocd-person/abc123"`), matching `member.id` directly

## API Endpoints Used

| Purpose | Endpoint | Format |
|---|---|---|
| Find bill + fetch all vote/summary/text data | `GET /bills?jurisdiction=ocd-jurisdiction/country:us/state:ga/government&identifier={id}&include=votes,abstracts,versions` | JSON |
| (Local file, not an API call) Party tally join | `assets/data/ga-members.json` | JSON |

All requests use the existing `OPENSTATES_API_KEY` secret. No additional credentials required.

Open States base URL: `https://v3.openstates.org`

## Vote Auto-Detection Logic

Each bill record includes a `votes` array of vote events. The workflow selects the final
passage vote per chamber using this logic:

1. Filter vote events to those with `motion_classification == ['passage']`
2. Filter to the target chamber (`organization.classification == 'upper'` for Senate, `'lower'` for House)
3. Parse the roll number from `motion_text` — format: `"House Vote #56 - 2025-2026 Regular Session"`
4. **Take the event with the highest roll number** (latest in session order)

If `voteOverride` is set in `curated-ga-bills.json`, the workflow skips auto-detection and uses that roll number directly.

**Why highest roll number?** GA sometimes records multiple `passage`-classified votes on the same
bill in the same chamber on the same day (e.g. a substitute amendment vote followed by final passage).
The higher number is always the later vote, which is final passage.

## Party Tally Derivation

Open States does not pre-compute party tallies. The workflow derives them at generation time:

```
for each PersonVote in vote_event.votes:
    look up voter.id in ga-members.json → get party
    increment partyTally[party][option]
```

Member party values from `ga-members.json`: `"Democrat"`, `"Republican"`, `"Independent"`.

Open States vote options: `"yes"`, `"no"`, `"abstain"`, `"other"` — map directly to
output keys. `"other"` covers not-voting, excused, and absent.

## Member Page Matching (Client-Side)

```javascript
// Both House and Senate GA members — direct OCD person ID match
const myVote = bill.votes.house?.memberVotes[member.id]
             ?? bill.votes.senate?.memberVotes[member.id];
```

No secondary resolution step needed. The `voter.id` field in Open States vote records
is the same OCD person ID format as `member.id` in `ga-members.json`.

## Workflow: `update-curated-ga-bills.yml`

New workflow, runs daily alongside the existing GA member data workflow.

```
Trigger: daily schedule + workflow_dispatch

Steps:
1. Read curated-ga-bills.json
2. Load ga-members.json → build id→party lookup map
3. For each bill:
   a. Search Open States: /bills?jurisdiction=ga&identifier={id}&include=votes,abstracts,versions
   b. Take first result → title, status, statusDate, openstatesUrl
   c. Extract summary from abstracts[0].abstract; use summaryOverride if set
   d. Extract fullTextUrl from last version's links (prefer most-recent note, take first PDF link)
   e. For house and senate:
      - Filter vote events: motion_classification=['passage'], correct chamber
      - Select by voteOverride roll number, or highest roll number (auto)
      - Derive partyTally by joining voter.id → party lookup map
      - Build memberVotes dict: { voter.id → option }
4. Write curated-ga-bill-votes.json
5. Commit and push
```

## Member Page Display

New section on `ga-member.html` below contact info, matching the Congressional layout.

**Bill card shows:**
- Title (linked to `openstatesUrl`)
- Summary (truncated with expand option if long)
- Status + date
- Member's vote — `Yes` / `No` / `Abstain` / `Not Voting` / `No vote recorded`
- Party tally bar — visual D vs R breakdown (yea votes)
- "Read full text (PDF)" link

**Display logic:**
```javascript
// Determine which chamber to show for this member
const chamberKey = (member.chamber === 'Senate') ? 'senate' : 'house';
const voteData   = bill.votes[chamberKey];
const myVote     = voteData?.memberVotes[member.id];
```

A senator only sees the Senate vote; a House member only sees the House vote.
Bills that haven't crossed to the member's chamber show "No vote recorded."

## Risks and Mitigations

| Risk | Mitigation |
|---|---|
| Two `passage` votes in same chamber on same day | Take highest roll number; `voteOverride` available if wrong |
| Open States GA data lags 1–2 days during session | Acceptable — daily build; show `statusDate` so users see data age |
| `voter.id` is null for some members | Fall back to `voter_name` match if id is null; log warning |
| Bill abstract is absent or too technical | `summaryOverride` in curated-ga-bills.json |
| Bill text is PDF-only | Label as "(PDF)" in the UI; no workaround needed |
| `motion_classification` is not `['passage']` for a vote we want | Use `voteOverride` with the explicit roll number |
| Open States rate limits | One request per curated bill; target 10–20 bills keeps well within limits |
| Session string wrong in curated list | Workflow logs the bill not found; easy to correct |

## Differences from the Congressional Feature

| | Congressional | Georgia |
|---|---|---|
| Data source | Congress.gov API + senate.gov XML | Open States v3 API |
| API key | `CONGRESS_API_KEY` | `OPENSTATES_API_KEY` (already used for members) |
| Member matching | `bioguideId` (House) / `last_name:state` (Senate) | OCD person ID — direct match, no resolution needed |
| Party tally | Pre-computed in API response | Derived by joining voter.id against ga-members.json |
| Vote selection | Action code lookup (`H37100`, `17000`) | Last `['passage']` vote event by roll number |
| Vote options | Aye / Nay / Present / Not Voting | yes / no / abstain / other |
| Bill text format | HTML (formatted text) | PDF only |
| Bill identifier | congress + type + number (e.g. `119/s/331`) | session + identifier (e.g. `2025-2026 / HB 90`) |
| Summary source | CRS summaries endpoint | `abstracts[0].abstract` on bill record |
